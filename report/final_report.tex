\section*{Задача №1. Проектирование Call-центра.}
Известно, что среднее время между звонками клиентов составляет Tc секунд, а среднее время обслуживания Ts секунд. Все потоки случайных событий считать пуассоновскими.

\noindent\textbf{1.} Рассмотреть систему без очереди. Построить графики от числа операторов: вероятности отказа вплоть до обеспечения отказов менее 1\%; математического ожидания числа занятых операторов; коэффициента загрузки операторов.

\noindent\textbf{Исходные данные:}\par
\smallskip
\noindent Tc = 20\par
\noindent Ts = 59\par

\begin{figure}[H]
\centering
\includegraphics[width=0.9\textwidth]{\detokenize{assets/task1_1__scheme.png}}
\caption*{Рисунок 1.1.1. Расчетная схема.}
\end{figure}
\noindent\textbf{Обозначения:}\par
\smallskip
\noindent S\_k — состояние с k занятыми операторами, k = 0, ..., n.\par
\noindent λ = 1/Tc = 0.05 1/с, μ = 1/Ts = 0.017 1/с.\par
\noindent a = λ/μ — offered load; p\_k — стационарная вероятность состояния S\_k.\par
\noindent P\_отк — вероятность отказа; M\_зан и K\_загр — среднее число занятых операторов и их загрузка.\par

\noindent\textbf{Переход от исходных данных к вероятностям состояний.} Сначала по Tc и Ts находим интенсивности потока и обслуживания, а затем выражаем через них стационарные вероятности состояний S\_k, где k равно числу занятых операторов.

\[
a = \lambda / \mu
\]

\[
p_0 = \left(\sum_{k=0}^{n}\frac{a^k}{k!}\right)^{-1}
\]

\[
p_k = \frac{a^k}{k!}p_0,\quad k=0,\ldots,n
\]
Для текущего варианта offered load равен 2.95, поэтому даже в системе без очереди крайнее состояние S\_n заметно влияет на итоговые метрики.

Далее из того же распределения последовательно получаются средняя занятость операторов и вероятность отказа.

\noindent\textbf{Занятые операторы и загрузка.} Сначала по найденным p\_k суммируем среднее число занятых операторов, а затем нормируем его по n и получаем коэффициент загрузки.

\[
M_{\mathrm{зан}} = \sum_{k=0}^{n}k\,p_k
\]

\[
K_{\mathrm{загр}} = M_{\mathrm{зан}}/n
\]
Числовой checkpoint: при n = 5 имеем M\_зан = 2.638 и K\_загр = 0.528; при n = 8 — 2.928 и 0.366.

\begin{figure}[H]
\centering
\includegraphics[width=0.8\textwidth]{\detokenize{../figures/task1_1__busy_operators_vs_operators.png}}
\caption*{Рисунок 1.1.2. Среднее число занятых операторов по числу операторов.}
\end{figure}
График показывает, что после n = 5 среднее число занятых операторов растёт уже слабо: добавление новых операторов дальше создаёт прежде всего запас по отказам, а не существенный прирост фактической занятости.

\noindent\textbf{Вероятность отказа.} Затем рассматриваем крайнее состояние S\_n: именно оно соответствует ситуации, когда новый звонок приходит в момент полной занятости всех операторов.

\[
P_{\mathrm{отк}} = p_n
\]
Числовой checkpoint: при n = 7 вероятность отказа ещё равна 0.02, а при n = 8 уже снижается до 0.007.

\begin{figure}[H]
\centering
\includegraphics[width=0.8\textwidth]{\detokenize{../figures/task1_1__refusal_and_utilization_vs_operators.png}}
\caption*{Рисунок 1.1.3. Вероятность отказа и коэффициент загрузки по числу операторов.}
\end{figure}
По графику отказов условие P\_отк < 0.01 впервые выполняется при n = 8.

В этой точке вероятность отказа равна 0.007, поэтому значение n = 8 принимается как минимально достаточное для текущего варианта.

\clearpage
\noindent\textbf{2.} Рассмотреть систему с ограниченной очередью. Варьируя число операторов вплоть до 15, построить семейства графиков от числа мест в очереди: вероятности отказа; математического ожидания числа занятых операторов; коэффициента загрузки операторов; вероятности существования очереди; математического ожидания длины очереди; коэффициента занятости мест в очереди. Варьируя число мест в очереди вплоть до 15, построить семейства графиков от числа операторов для тех же метрик.

\noindent\textbf{Исходные данные:}\par
\smallskip
\noindent Tc = 20\par
\noindent Ts = 59\par

\begin{figure}[H]
\centering
\includegraphics[width=0.9\textwidth]{\detokenize{assets/task1_2__scheme.png}}
\caption*{Рисунок 1.2.1. Расчетная схема.}
\end{figure}
\noindent\textbf{Обозначения:}\par
\smallskip
\noindent S\_k при k = 0, ..., n описывают число занятых операторов; правые состояния отвечают длине очереди r = 1, ..., m.\par
\noindent λ = 0.05 1/с, μ = 0.017 1/с; a = λ/μ.\par
\noindent ρ\_n = λ/(nμ) — коэффициент загрузки системы при фиксированном n; p\_k — вероятности состояний.\par
\noindent P\_отк — вероятность отказа; P\_оч — вероятность существования очереди; L\_оч и K\_мест — средняя длина и занятость мест очереди.\par
\noindent M\_зан — среднее число занятых операторов; K\_загр — коэффициент их загрузки.\par

\noindent\textbf{Нормировка распределения.} После состояний обслуживания добавляются состояния ожидания, поэтому сначала выписываем offered load, коэффициент ρ\_n и нормировочную формулу p\_0.

\[
a = \lambda / \mu,\quad \rho_n = \lambda /(n\mu)
\]

\[
p_0 = \left(\sum_{k=0}^{n}\frac{a^k}{k!} + \frac{a^n}{n!}\sum_{r=1}^{m}\rho_n^{\,r}\right)^{-1}
\]
На этом шаге вся вероятность уже делится между занятыми операторами и правым хвостом очереди, поэтому одна и та же нормировка дальше обслуживает и потери, и показатели ожидания.

\noindent\textbf{Вероятности состояний обслуживания и очереди.} Затем разделяем вероятности на две части: состояния k = 0, ..., n описывают обслуживание, а состояния n + r — очередь длины r.

\[
p_k = \frac{a^k}{k!}p_0,\quad k=0,\ldots,n
\]

\[
p_{n+r} = \frac{a^n}{n!}\rho_n^{\,r}p_0,\quad r=1,\ldots,m
\]
Именно эти p\_k затем используются во всех семействах графиков без ручной подстановки отдельных режимов.

Для текущего варианта просматриваются все сочетания n = 1..15 и m = 1..15, поэтому сравнение по графикам остаётся полным и последовательным.

\noindent\textbf{Семейства по числу мест в очереди.} Сначала фиксируем число операторов n и смотрим, как изменение m перераспределяет потери и ожидание.

\noindent\textbf{Вероятность отказа.} При фиксированном n отказ определяется последним состоянием цепи с заполненной очередью.

\[
P_{\mathrm{отк}} = p_{n+m}
\]
Числовой checkpoint: при n = 5 рост m от 1 до 5 снижает P\_отк с 0.059 до 0.007, а при m = 15 вероятность отказа становится < 0.001.

\begin{figure}[H]
\centering
\includegraphics[width=0.8\textwidth]{\detokenize{../figures/task1_2__refusal_vs_queue__family_by_operators.png}}
\caption*{Рисунок 1.2.2. Вероятность отказа по числу мест в очереди при разных n.}
\end{figure}
Левое семейство подтверждает, что первые дополнительные места в очереди дают наибольший выигрыш по потерям, после чего эффект постепенно насыщается.

\noindent\textbf{Занятые операторы и загрузка.} По тем же вероятностям состояний отдельно вычисляем среднее число занятых операторов и коэффициент их загрузки.

\[
M_{\mathrm{зан}} = \sum_{k=0}^{n+m}\min(k,n)\,p_k
\]
При том же n = 5 среднее число занятых операторов увеличивается лишь с 2.777 до 2.95, а коэффициент загрузки — с 0.555 до 0.59.

Следовательно, расширение очереди почти не создаёт новый ресурс обслуживания, а прежде всего уменьшает долю потерянных вызовов.

\noindent\textbf{Очередь.} Теперь переходим к метрикам ожидания: они показывают, какая часть ранее потерянных вызовов переходит в очередь.

\[
P_{\mathrm{оч}} = \sum_{k=n+1}^{n+m}p_k
\]

\[
L_{\mathrm{оч}} = \sum_{k=n+1}^{n+m}(k-n)\,p_k,\quad K_{\mathrm{мест}} = L_{\mathrm{оч}}/m
\]
При n = 5 переход от m = 1 к m = 15 увеличивает вероятность существования очереди с 0.059 до 0.132, а среднюю длину очереди — с 0.059 до 0.321.

Одновременно занятость одного места падает с 0.059 до 0.021, то есть очередь становится длиннее, но каждое отдельное место используется менее интенсивно.

\noindent\textbf{Вероятность отказа.} При фиксированном m используется то же выражение для P\_отк, но теперь основной фактор — увеличение доступного числа операторов.

Числовой checkpoint: при m = 5 увеличение числа операторов с n = 5 до n = 8 снижает P\_отк с 0.007 до < 0.001; при n = 11 оно остаётся на уровне < 0.001.

\noindent\textbf{3.} Рассмотреть систему без ограничений на длину очереди. Построить графики от числа операторов вплоть до 15: математического ожидания числа занятых операторов; коэффициента загрузки операторов; вероятности существования очереди; математического ожидания длины очереди.

\noindent\textbf{Исходные данные:}\par
\smallskip
\noindent Tc = 20\par
\noindent Ts = 59\par

\begin{figure}[H]
\centering
\includegraphics[width=0.9\textwidth]{\detokenize{assets/task1_3__scheme.png}}
\caption*{Рисунок 1.3.1. Расчетная схема.}
\end{figure}
\noindent\textbf{Обозначения:}\par
\smallskip
\noindent Состояния до границы n описывают занятых операторов, а правый хвост схемы — число ожидающих заявок.\par
\noindent λ = 0.05 1/с, μ = 0.017 1/с; a = λ/μ.\par
\noindent ρ\_n = λ/(nμ) — условие стационарности; стационарные метрики вычисляются только при ρ\_n < 1.\par
\noindent P\_wait — вероятность того, что поступившая заявка застала все n операторов занятыми.\par
\noindent P\_оч и L\_оч — вероятность существования очереди и средняя длина очереди в стационарном режиме.\par

\noindent\textbf{Проверка стационарности.} Для бесконечной очереди вычисления начинаются с условия ρ\_n < 1: без него стационарное распределение и все средние характеристики просто не существуют.

\[
a = \lambda / \mu,\quad \rho_n = \lambda /(n\mu)
\]

\[
p_0 = \left(\sum_{k=0}^{n-1}\frac{a^k}{k!} + \frac{a^n}{n!(1-\rho_n)}\right)^{-1},\quad \rho_n < 1
\]
Для текущего варианта a = 2.95, поэтому условие ρ\_n < 1 впервые выполняется только при n = 3.

Следовательно, режимы n = 1 и n = 2 остаются нестационарными и в графиках не получают выдуманных стационарных значений.

\noindent\textbf{Вероятность ожидания.} После нормировки можно отдельно выписать вероятность того, что новая заявка застанет все n операторов занятыми; именно через неё дальше выражаются метрики очереди.

\[
P_{\mathrm{wait}} = \frac{a^n}{n!(1-\rho_n)}p_0
\]
В первой стационарной точке n = 3 получаем P\_wait = 0.969, то есть почти каждая новая заявка попадает в ожидание.

\noindent\textbf{4.} Рассмотреть систему без ограничений на длину очереди, учитывающую фактор ухода клиентов из очереди при среднем приемлемом времени ожидания Tw. Построить графики от числа операторов вплоть до 15: математического ожидания числа занятых операторов; коэффициента загрузки операторов; вероятности существования очереди; математического ожидания длины очереди.

\noindent\textbf{Исходные данные:}\par
\smallskip
\noindent Tc = 20\par
\noindent Ts = 59\par
\noindent Tw = 103\par

\begin{figure}[H]
\centering
\includegraphics[width=0.9\textwidth]{\detokenize{assets/task1_4__scheme.png}}
\caption*{Рисунок 1.4.1. Расчетная схема.}
\end{figure}
\noindent\textbf{Обозначения:}\par
\smallskip
\noindent S\_k описывают общее число заявок в системе; при k > n часть заявок находится в очереди.\par
\noindent λ = 0.05 1/с, μ = 0.017 1/с, ν = 1/Tw = 0.01 1/с.\par
\noindent β\_k — интенсивность поступления новых заявок; δ\_k — суммарная интенсивность обратного перехода из состояния k.\par
\noindent p\_k — стационарная вероятность состояния после численного усечения хвоста.\par
\noindent P\_оч и L\_оч — вероятность существования очереди и средняя длина очереди с учётом ухода клиентов.\par

\noindent\textbf{Интенсивности переходов.} В этой модели поток поступления остаётся тем же, но после заполнения операторов обратные переходы ускоряются за счёт ухода клиентов из очереди.

\[
\nu = 1/T_w,\quad \beta_k = \lambda
\]

\[
\delta_k = \min(k,n)\mu + \max(k-n,0)\nu
\]
Пока k ≤ n, выход из состояния определяется только обслуживанием. После появления очереди к нему добавляется уход клиентов со скоростью ν, поэтому дальний хвост распределения затухает быстрее, чем без ухода.

\noindent\textbf{Рекуррентное построение и усечение хвоста.} После задания β\_k и δ\_k вероятности p\_k строятся рекуррентно. Поскольку хвост бесконечен, численное суммирование обрывается только после достижения заданного порога точности.

\[
p_k = p_{k-1}\beta_k/\delta_k,\quad k \ge 1
\]

\[
p_0 = \left(1 + \sum_{k=1}^{\infty}\prod_{i=1}^{k}\beta_i/\delta_i\right)^{-1}
\]
Во всём диапазоне n = 1..15 бесконечный хвост распределения усечён при epsilon = 1e-12.

Оставшийся вклад не превышает 8.465e-14 по вероятности и 9.464e-13 по среднему числу заявок в очереди, поэтому он пренебрежим в пределах учебной точности отчёта.

Во всех рассчитанных точках сохраняется статус stationary\_truncated, то есть стационарные метрики получены после контролируемого численного усечения, а не после произвольной отсечки.

\noindent\textbf{Занятые операторы.} После нормировки суммируем среднее число занятых операторов и по нему оцениваем загрузку каждого оператора.

\[
M_{\mathrm{зан}} = \sum_{k=0}^{\infty}\min(k,n)\,p_k,\quad K_{\mathrm{загр}} = M_{\mathrm{зан}}/n
\]
Числовой checkpoint: при n = 1 получаем M\_зан = 0.978 и K\_загр = 0.978; при n = 5 — 2.857 и 0.571; при n = 8 — 2.947 и 0.368.

\noindent\textbf{Очередь и уход клиентов.} Теперь рассматриваем хвост состояний k > n: именно он определяет вероятность существования очереди и её среднюю длину с учётом ухода клиентов.

\section*{Задача №2. Проектирование производственного участка.}
Имеется участок с N станками. Среднее время между наладками составляет Tc минут, среднее время наладки — Ts минут. Все потоки случайных событий считать пуассоновскими.

Построить графики от числа наладчиков: математического ожидания числа простаивающих станков; математического ожидания числа станков, ожидающих обслуживания; вероятности ожидания обслуживания; математического ожидания числа занятых наладчиков; коэффициента занятости наладчиков.

\noindent\textbf{Исходные данные:}\par
\smallskip
\noindent N = 33\par
\noindent Tc = 110\par
\noindent Ts = 44\par

\centering

\end{figure}
\noindent\textbf{Обозначения:}\par
\smallskip
\noindent S\_i — состояние с i неисправными станками.\par
\noindent λ = 1/Tc = 0.009 1/мин, μ = 1/Ts = 0.023 1/мин.\par
\noindent λ\_i = (N-i)λ — интенсивность очередного отказа; μ\_i = min(i,r)μ — интенсивность восстановления.\par
\noindent p\_i — стационарная вероятность состояния S\_i.\par
\noindent M\_пр и M\_ож — средние числа простаивающих и ожидающих обслуживания станков; P\_ож — вероятность того, что новый отказавший станок попадёт в ожидание.\par
\noindent M\_зан и K\_загр — среднее число занятых наладчиков и коэффициент их занятости.\par

\noindent\textbf{Распределение состояний.} Вероятности состояний строятся по числу неисправных станков i, а интенсивности отказов и восстановления меняются вместе с текущим состоянием участка.

\[
\lambda_i = (N-i)\lambda,\quad \mu_i = \min(i,r)\mu
\]

\[
p_i = p_{i-1}\lambda_{i-1}/\mu_i,\quad i=1,\ldots,N
\]

\[
p_0 = \left(1 + \sum_{i=1}^{N}\prod_{j=1}^{i}\lambda_{j-1}/\mu_j\right)^{-1}
\]
\noindent\textbf{Простаивающие и ожидающие станки.} Сначала по распределению состояний вычисляются средние числа простаивающих станков и станков, ожидающих обслуживания.

\[
M_{\mathrm{пр}} = \sum_{i=0}^{N}i\,p_i,\quad M_{\mathrm{ож}} = \sum_{i=0}^{N}\max(i-r,0)\,p_i
\]

\[
P_{\mathrm{ож}} = \frac{\sum_{i=r}^{N-1}(N-i)p_i}{\sum_{i=0}^{N-1}(N-i)p_i}
