# Контракт итогового отчёта

## Назначение
Этот документ фиксирует каноническую структуру финального отчёта и правила отделения вычисленных фактов от автоматически или вручную сгенерированного текста.

## Каноническая структура отчёта
1. Титульный лист в академическом стиле, совместимом с `references/DZ1.docx`.
2. `Задача №1. Проектирование Call-центра.`
3. Подраздел `1.1. Система без очереди`.
4. Подраздел `1.2. Система с ограниченной очередью`.
5. Подраздел `1.3. Система с неограниченной очередью`.
6. Подраздел `1.4. Система с неограниченной очередью и уходом клиентов`.
7. `Задача №2. Проектирование производственного участка.`
8. Подраздел `2.1. Метрики производственного участка по числу наладчиков`.

## Внутренняя структура каждого содержательного подраздела
Каждый из разделов `1.1`, `1.2`, `1.3`, `1.4`, `2.1` должен содержать в следующем порядке:
1. Краткую формулировку подпункта.
2. Исходные данные для текущей модели.
3. Расчётную схему.
4. Формулы вероятностей состояний.
5. Формулы производных метрик.
6. Рисунки и краткие пояснения к характерным режимам.

## Минимальный ожидаемый набор рисунков
Если финальный отчёт следует доступному образцу без упрощений, минимальный набор рисунков таков:
- Задача `1.1`: `3` рисунка.
- Задача `1.2`: `13` рисунков.
- Задача `1.3`: `5` рисунков.
- Задача `1.4`: `5` рисунков.
- Задача `2.1`: `6` рисунков.
- Итого: `32` рисунка.

## Ожидаемые вычислимые артефакты
- Машиночитаемые таблицы метрик для каждого sweep-сценария.
- Отрисованные графики из этих таблиц.
- Зафиксированные формулы/обозначения для каждого подпункта.
- Текстовые фрагменты интерпретации результатов.

## Рекомендуемая раскладка артефактов
- `artifacts/data/` для CSV/JSON с вычисленными точками.
- `artifacts/figures/` для PNG/SVG/PDF графиков и расчётных схем.
- `artifacts/formulas/` для Markdown/LaTeX/OMML-совместимых формульных фрагментов.
- `artifacts/report/` для собираемого отчёта и вспомогательных промежуточных представлений.

Эти каталоги Stage 01 не создаёт; документ только фиксирует контракт для следующих стадий.

## Правило разделения вычисленных фактов и пояснительного текста
- В вычисленные факты входят:
  - подтверждённые входные параметры варианта;
  - все рассчитанные значения метрик;
  - таблицы sweep-ов;
  - правила отсечения нестабильных режимов;
  - значения, попавшие на графики;
  - формульные определения.
- В пояснительный текст входят:
  - вводные фразы;
  - связующий текст между формулами и рисунками;
  - интерпретации поведения кривых;
  - выводы по режимам.
- Пояснительный текст не имеет права вводить новые числа, которых нет в машинно-читаемых данных.
- Любое числовое утверждение в тексте должно быть воспроизводимо из `artifacts/data/`.
- Любой график должен иметь однозначный источник в `artifacts/data/` и однозначное имя в `artifacts/figures/`.
- Нестабильные/неприменимые точки должны явно помечаться как таковые, а не маскироваться «красивым» текстом.

- По доступным входам обязательные таблицы не выявлены.
