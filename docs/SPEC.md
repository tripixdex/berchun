# Спецификация проекта

## Статус
Базовая спецификация была заморожена на `STAGE 01 — Audit Inputs + Freeze Spec` на основании локально доступных источников, перечисленных в `docs/AUDIT.md`. На `STAGE 02` подтверждён конкретный вариант и реализован аналитический вычислительный контур; это не меняет саму постановку задачи, но снимает прежний блокер по отсутствию числовых значений варианта.

## Точная цель репозитория
Этот репозиторий должен привести к воспроизводимому решению индивидуального варианта домашнего задания по курсу «Имитационное моделирование технологических производственных процессов» и к академически оформленному отчёту на русском языке по двум задачам:
- задача 1: проектирование call-центра;
- задача 2: проектирование производственного участка.

## Границы проекта
- В объём проекта входят только задачи и подпункты, явно перечисленные в `my_var.md`.
- Репозиторий должен быть параметризуемым: никакие числа из чужого примера не могут быть зашиты как значения нашего варианта.
- Основной целевой результат проекта: вычислимые метрики, графики и итоговый отчёт по своему варианту.
- В Stage 01 не входят реализация вычислителя, генерация графиков, сборка итогового отчёта и загрузка внешних материалов.
- В текущих входах нет подтверждения, что обязательна конкретная программная среда вроде GPSS; следовательно, инструментальная реализация остаётся подчинённой содержанию, а не наоборот.

## Что означает «идеальное завершение моего варианта»
Идеальное завершение означает, что в репозитории можно воспроизводимо получить:
1. Подтверждённые исходные параметры варианта: номер по журналу, день рождения, месяц рождения.
2. Все обязательные метрики для обеих задач на нужных диапазонах параметров.
3. Машиночитаемые данные, из которых построены все графики.
4. Академически оформленный отчёт на русском языке с титульным листом, расчётными схемами, формулами и рисунками, согласованный с образцом по дисциплине оформления.
5. Явную фиксацию всех допущений, нестабильных режимов и границ применимости формул.

## Замороженный вычислительный подход
- Базовый путь реализации должен быть аналитическим/детерминированным, а не имитационным по Монте-Карло.
- Основание для заморозки:
  - образец `references/DZ1.docx` оформлен через формулы состояний и аналитические графики;
  - постановка задания требует стационарные метрики и графики, но не требует статистического эксперимента;
  - аналитический расчёт лучше воспроизводится и не вносит случайный шум.
- Симуляционный код допускается только позже и только как вспомогательная проверка, но не как замена обязательного аналитического контура без отдельного решения.

## Требуемые выходы по задаче 1

### 1.1. Система без очереди
Обязательные содержательные результаты:
- расчётная схема;
- график математического ожидания числа занятых операторов от числа операторов;
- график или совмещённый график вероятности отказа и коэффициента загрузки операторов от числа операторов;
- фиксация минимального числа операторов, при котором вероятность отказа становится меньше `1%`.

Замороженная политика перебора:
- число операторов: натуральные значения, начиная с `1`, до первого достижения `P_refusal < 0.01`.

### 1.2. Система с ограниченной очередью
Обязательные содержательные результаты:
- расчётная схема;
- `6` семейств графиков от числа мест в очереди при различных числах операторов:
  - вероятность отказа;
  - математическое ожидание числа занятых операторов;
  - коэффициент загрузки операторов;
  - вероятность существования очереди;
  - математическое ожидание длины очереди;
  - коэффициент занятости мест в очереди;
- `6` семейств графиков от числа операторов при различных числах мест в очереди с тем же набором метрик.

Замороженная политика перебора:
- число операторов: `1..15` как явно заданный диапазон;
- число мест в очереди: `1..15` как замороженный вывод из формулировки «с ограниченной очередью» и из того факта, что случай без очереди уже выделен отдельно в подпункт `1.1`.

### 1.3. Система с неограниченной очередью
Обязательные содержательные результаты:
- расчётная схема;
- график математического ожидания числа занятых операторов от числа операторов;
- график коэффициента загрузки операторов от числа операторов;
- график вероятности существования очереди от числа операторов;
- график математического ожидания длины очереди от числа операторов;
- явное указание области, где стационарный режим отсутствует и бесконечная очередь не имеет конечного стационарного среднего.

Замороженная политика перебора:
- число операторов: `1..15`.

### 1.4. Система с неограниченной очередью и уходом клиентов
Обязательные содержательные результаты:
- расчётная схема;
- график математического ожидания числа занятых операторов от числа операторов;
- график коэффициента загрузки операторов от числа операторов;
- график вероятности существования очереди от числа операторов;
- график математического ожидания длины очереди от числа операторов;
- явное определение интенсивности ухода из очереди через `Tw`;
- явное описание численного правила, если в формуле возникает бесконечный ряд/сумма и требуется усечение.

Замороженная политика перебора:
- число операторов: `1..15`.

## Требуемые выходы по задаче 2
Обязательные содержательные результаты:
- расчётная схема;
- график математического ожидания числа простаивающих станков от числа наладчиков;
- график математического ожидания числа станков, ожидающих обслуживания, от числа наладчиков;
- график вероятности ожидания обслуживания от числа наладчиков;
- график математического ожидания числа занятых наладчиков от числа наладчиков;
- график коэффициента занятости наладчиков от числа наладчиков.

Замороженная политика перебора:
- число наладчиков: `1..N`.
- Статус этого правила: вывод, а не прямой текст постановки.
- Обоснование: в системе с `N` станками значения числа наладчиков больше `N` физически избыточны и не дают нового режима.

## Обязательные формулы и разделы, если судить по доступным образцам
- Для каждого подпункта должен быть раздел с исходными данными.
- Для каждого подпункта должна быть расчётная схема.
- Для каждого подпункта должны присутствовать формулы вероятностей состояний и формулы перехода от вероятностей состояний к итоговым метрикам.
- Для задачи 1.3 должен быть отдельный комментарий о стационарности/нестационарности.
- Для задачи 1.4 должен быть отдельный комментарий о модели ухода и о численной сходимости.
- Для задачи 2 должен быть выписан конечный набор состояний от `0` до `N`, что подтверждается образцом.
- Обязательные таблицы из доступных входов не следуют; основной формат итогового содержимого состоит из формул и рисунков.

## Критерии качества
- Все исходные параметры варианта подтверждены и отделены от производных параметров.
- Ни один итоговый график не строится из вручную набранных чисел; у каждого графика есть машинный источник данных.
- Все вероятности находятся в диапазоне `[0, 1]`.
- Все математические ожидания ограничены физическим смыслом модели:
  - число занятых операторов не больше числа операторов;
  - длина очереди не отрицательна;
  - число простаивающих станков и число ожидающих станков не превышают `N`.
- Нестабильные или неприменимые точки не маскируются фиктивными числами.
- Чужие примерные значения из `references/DZ1.docx` не используются как наши результаты.
- Текст отчёта отделён от вычисленных фактов и ссылается на конкретные артефакты.
