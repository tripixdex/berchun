# Methodical Defense Template

## Статус и назначение
Этот документ замораживает repeatable defense-note template для methodical guide.

Template нужен для того, чтобы future guide:
- помогал устной защите;
- оставался variant-aware;
- не превращался в набор случайных советов.

Здесь фиксируется только структура карточек защиты, без наполнения финальной prose.

## Canonical defense card
Для каждого защитного micro-block используется один и тот же шаблон:

- `Подпункт`
- `Тема карточки`
- `Артефакт-опора`
- `Возможный вопрос преподавателя`
- `Короткий безопасный ответ`
- `Где студент обычно путается`
- `Запрещённая небрежная формулировка`
- `Практический смысл в одной строке`

## Required meaning of each field
- `Подпункт`: один из `1.1`, `1.2`, `1.3`, `1.4`, `2.1`.
- `Тема карточки`: что именно защищается — модель, формула, график, checkpoint, вывод.
- `Артефакт-опора`: конкретный data field, figure id или selector-backed checkpoint.
- `Возможный вопрос преподавателя`: короткий и реалистичный вопрос, который можно задать по текущему подпункту.
- `Короткий безопасный ответ`: одна-две фразы без лишней импровизации.
- `Где студент обычно путается`: главное место риска, которое guide должен подсветить.
- `Запрещённая небрежная формулировка`: опасный жаргон, ложное упрощение или технически неверный пересказ.
- `Практический смысл в одной строке`: ответ на вопрос «что это означает для реальной системы».

## Minimal card set per subsection
Для каждого подпункта future guide должен иметь минимум `4` defense cards:
1. Карточка `смысл модели`.
2. Карточка `смысл формулы или метрики`.
3. Карточка `как читать график`.
4. Карточка `что означает итоговый вывод`.

Если в подпункте есть особый риск путаницы, допускается обязательная `5`-я карточка:
- `ограничение / чего нельзя говорить`.

## Card usage rules
- Каждая карточка должна ссылаться на реальный артефакт, а не на «общую память».
- Если в ответе используется число, оно должно быть pulled from artifact map, а не сочинено вручную.
- `Короткий безопасный ответ` не должен превращаться в длинный абзац лекции.
- `Запрещённая небрежная формулировка` должна быть конкретной, а не абстрактной.
- `Практический смысл в одной строке` должен переводить математику в реальный смысл системы.

## Canonical blank template
Ниже замораживается пустая форма, которую M2 сможет заполнять variant-aware content:

```md
### Defense Card
- Подпункт:
- Тема карточки:
- Артефакт-опора:
- Возможный вопрос преподавателя:
- Короткий безопасный ответ:
- Где студент обычно путается:
- Запрещённая небрежная формулировка:
- Практический смысл в одной строке:
```

## What this template does NOT allow
- Не разрешает писать финальные ответы без artifact anchor.
- Не разрешает превращать защиту в отдельный appendix вне подпункта.
- Не разрешает использовать один общий шаблонный ответ для всех задач.
- Не разрешает подменять безопасный ответ жаргоном или неподтверждёнными числами.
